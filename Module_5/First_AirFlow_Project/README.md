**Постановка задачи**  
Разработать единый, воспроизводимый пайплайн для формирования батч-признаков, который обеспечит их идентичный расчет как на этапе обучения ML-модели, так и при инференсе  

**Окружение и зависимости**  
  1. Python 3.9 - 3.11  
  2. Apache Airflow с установленными провайдерами  
    - apache-airflow-providers-postgres  
    - apache-airflow-providers-amazon  

Дополнительные библиотеки:  
  - pandas
  - sqlalchemy
  - boto3
  - botocore

В проекте должен быть файл calculate_batch_features.py с функцями table_preprocessing и calculate_features.  

DAG назвается batch_features  
Запустить можно вручную через веб-интерфейс Airflow. Автоматического расписания нет - только ручной запуск  
Параметр run_date читается из Airflow Variables  

**Переменные и подключения**  
Переменные Airflow:  
  - Имя: batch_features_run_date
  - Описание: дата, на которую рассчитывается батч-признаки
  - Тип: String
  - Пример значения: 2025-10-01 

Подключения:
  - postgres_raw : Исходная БД с таблицами customers, sessions, events, orders
  - s3_features : Целевое S3-хранилище для сохранения CSV

**Сохранение результата**  
  - Результат формируется в виде CSV-файла и загружается в S3 по ключу "run_date=<дата запуска>/batch_features.csv"  
  - DAG содержит встроенную задачу validate_saved_result, которая автоматически проверяет:
    1. Что количество строк в загруженном файле больше нуля
    2. Что S3-объект действительно существует
